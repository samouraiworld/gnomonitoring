## 1. Critical Issues (Must Fix Before Merge)

### 🔴 C1. Shared `lastRPCErrorAlert` / `lastProgressTime` across all chains

**Severity:** CRITICAL (Correctness bug)
**File:** `internal/gnovalidator/gnovalidator_realtime.go`, lines 22-24
**Impact:** Cross-chain alert suppression

`lastRPCErrorAlert` and `lastProgressTime` are global singletons shared across all chains. When running N chains, an RPC error on chain A updates the global `lastRPCErrorAlert`, suppressing the anti-spam guard for chain B for 10 minutes.

**Fix:** Convert to per-chain maps like `lastProgressHeight`

---

### 🔴 C2. `GetMonikerMap` returns live map reference (DATA RACE)

**Severity:** CRITICAL (Race condition)
**File:** `internal/gnovalidator/gnovalidator_realtime.go`, lines 600-607
**Impact:** Concurrent map modification

After `RUnlock()`, the caller holds an unprotected pointer to the map. Meanwhile `SetMoniker` (which takes a write lock) can concurrently modify it.

**Fix:** Return a shallow copy inside the read lock

---

### 🔴 C3. `InsertWebhook` uses type-unsafe variadic arguments

**Severity:** CRITICAL (Type safety)
**File:** `internal/database/db.go`, lines 42-65
**Impact:** Silent failures, nil dereferences

Type checking is done at runtime with `switch`. If a caller passes wrong types, it silently fails.

**Fix:** Use explicit parameters instead of variadic `...interface{}`

---

### 🔴 C4. Missing-series CTE duplicated in three places

**Severity:** CRITICAL (Code duplication)
**Files:** View definition + two inline copies in `gnovalidator_realtime.go`

Three independently-maintained versions of the same complex query. If one is fixed but others are not, alert behavior diverges.

**Fix:** Use the view consistently, remove inline copies

---

### 🔴 C5. `AlertLog` deduplication broken at DB level

**Severity:** CRITICAL (Logic error)
**File:** `internal/database/db_metrics.go`, line 26
**Impact:** Alert duplication

With auto-increment `id` as PK, the `OnConflict{DoNothing: true}` never detects duplicates. Deduplication breaks.

**Fix:** Restore composite PK constraint for `(addr, level, start_height, end_height)`
